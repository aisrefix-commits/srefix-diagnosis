# Reporting Copyright / Content Concerns

This project's diagnostic content (`agents/*.md`) was synthesized by Large
Language Models and validated against publicly-available open-source
documentation. We took care to paraphrase rather than copy, but if you
believe any specific content reproduces copyrighted text verbatim or
otherwise infringes your rights, please follow the process below.

## Quick takedown procedure

1. **Open a GitHub Issue** in this repository titled `[copyright] <tech-name>`
   (replace `<tech-name>` with the affected file, e.g. `postgres-agent.md`).

2. Include in the issue:
   - The file path and line range (e.g. `agents/postgres-agent.md:120-145`)
   - The specific phrase or paragraph you believe is infringing
   - The original public source you believe was copied (URL + snippet)
   - Your relationship to the copyrighted material (rights holder /
     authorized agent / project maintainer)

3. We aim to respond within **5 business days**, and to remove or rewrite
   the contested content within **14 business days** of confirmation. If a
   takedown is more urgent (e.g. legal hold), email the maintainers
   directly (see `MAINTAINERS.md` if present, otherwise via the GitHub
   profile).

4. We will:
   - Compare the cited public source to the file content
   - Remove the section if substantial similarity is confirmed
   - Note the change in `CHANGELOG.md` with reason "removed per copyright report"
   - Avoid disclosing the reporter's identity beyond what's already public

## What we ask of reporters

- One report per file (don't bundle multiple files in one issue — slows triage)
- A specific, line-anchored claim — "vibe of section X" is not actionable
- Public source URL when possible (helps us verify provenance)
- Good-faith assertion that you're authorized to make the claim

## What you should NOT report here

- General style or factual overlap with a tech's documentation: SRE
  knowledge about a database is shared across the industry; describing
  "PostgreSQL replication lag is measured by `pg_replication_lag_seconds`"
  reproduces a fact, not protected expression.
- Tradenames / trademarks: nominative use is permitted.
- Disagreement with an opinion: open a normal issue, not a copyright issue.

## How to verify content was LLM-synthesized

- Each markdown file under `agents/` was generated by Claude or GPT-class
  models, prompted with the technology's name and operational scope.
- The synthesis prompts asked the model to describe symptoms, diagnostic
  metrics, root-cause decision trees, and thresholds — without quoting
  source documentation.
- Validation steps (manual review + automated cross-check against public
  exporter metric names) were applied before commit. They are not perfect.

## Preventing future issues

If you maintain documentation for one of the technologies covered:

- Tell us so we can credit your project explicitly in `NOTICE`.
- Send a list of phrases/figures you'd like us to AVOID paraphrasing
  closely, and we'll add them to the synthesis-time exclusion list.
